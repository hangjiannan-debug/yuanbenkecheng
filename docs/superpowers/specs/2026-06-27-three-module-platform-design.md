# 园本课程建设平台 · 三版块架构设计

**日期**: 2026-06-27  
**状态**: 待评审

---

## 一、概述

将现有单页工具升级为三版块平台：园本课程建设（园长）、课程生成（教师）、我的园本课程（园长+教师）。引入账号体系，数据按园所隔离，纯前端 IndexedDB 存储。

---

## 二、账号体系

### 2.1 角色与权限

| 角色 | 可见版块 | 特殊权限 |
|------|---------|---------|
| 园长 | 全部三个版块 | 创建/管理教师账号，生成园本课程方案 |
| 教师 | 课程生成 + 我的园本课程 | 生成课程内容，上传实施记录 |

### 2.2 账号模型

- 园长注册：园所名称 + 手机号 + 密码 → 创建园所
- 园长在「账号管理」中创建教师账号（姓名 + 手机号 + 初始密码）
- 教师用分配的账号登录，自动绑定到对应园所
- 同一园所内数据共享（方案、课程、记录）

### 2.3 存储

IndexedDB 表 `accounts`：
```
{ id, role: 'principal'|'teacher', name, phone, password, kinderId, createdAt }
```

IndexedDB 表 `kinders`：
```
{ id, name, createdAt }
```

## 五、版块二：课程生成

### 5.1 入口

教师登录后，默认进入「课程生成」版块。

### 5.2 流程

```
选择方案 → 选择主题/年龄段 → 选择生成类型 → AI生成 → 预览编辑 → 保存
```

### 5.3 生成类型

**A. 周教学计划**
- 输入：方案、主题、年龄段、周次
- 输出：每日活动安排表
  - 活动名称、目标（认知/技能/情感）
  - 材料准备、活动流程（导入→展开→结束）
  - 延伸建议、观察要点

**B. 主题资源包**
- 输入：方案、主题、年龄段
- 输出：
  - 绘本推荐（3-5本，含推荐理由）
  - 游戏方案（3-5个，含玩法说明）
  - 环境创设建议（区域布置、材料投放）
  - 家园共育任务（亲子活动建议）

### 5.4 生成逻辑

基于方案中的主题词和年龄段，通过预设模板 + 动态内容填充生成。不依赖外部API。

### 5.5 数据存储

IndexedDB 表 `course_contents`：
```
{
  id, planId, kinderId, theme, ageGroup,
  type: 'weekly_plan' | 'resource_pack',
  content: {},            // 结构化内容
  htmlContent: '',
  createdBy, createdAt, updatedAt
}
```

## 六、版块三：我的园本课程

### 6.1 布局

```
┌──────────────────────────────────────────────────┐
│ 左侧：方案/课程列表         │ 右侧：详情 + 记录区  │
│ ┌────────────────────┐    │ ┌──────────────────┐ │
│ │ 🔍 搜索...         │    │ │ 方案详情          │ │
│ │                    │    │ │ [编辑] [下载PDF]  │ │
│ │ 📋 活力童年方案     │    │ │ [打印]            │ │
│ │  └ 周计划-小班     │    │ │                  │ │
│ │  └ 资源包-小班     │    │ │ ...              │ │
│ │  └ 实施记录(5)     │    │ ├──────────────────┤ │
│ │                    │    │ │ 关联课程内容       │ │
│ │ 📋 艺韵童心方案     │    │ │ 周计划 / 资源包   │ │
│ │                    │    │ ├──────────────────┤ │
│ └────────────────────┘    │ │ 实施记录          │ │
│                           │ │ [筛选: 全部▼]    │ │
│                           │ │ [+ 上传记录]     │ │
│                           │ │ 作品 / 观察 /    │ │
│                           │ │ 反思 / 照片 /    │ │
│                           │ │ 视频             │ │
│                           │ └──────────────────┘ │
└──────────────────────────────────────────────────┘
```

### 6.2 方案管理

- 列表展示所有已生成的方案（按时间倒序）
- 点击方案 → 右侧展示方案详情（完整HTML渲染）
- 支持操作：编辑（手动修改文字）、下载PDF（window.print）、打印

### 6.3 课程内容管理

- 方案下展示关联的课程内容（教案、资源包）
- 可查看、编辑、删除

### 6.4 实施记录

**上传类型**：
| 类型 | 格式 | 说明 |
|------|------|------|
| 幼儿作品 | 图片 | 拍照上传，可附文字说明 |
| 观察记录 | 文字 | 结构化模板（观察对象、时间、场景、行为描述、分析、回应） |
| 教学反思 | 文字 | 自由文本 |
| 照片 | 图片 | 活动照片，可批量上传 |
| 视频 | 视频 | 活动视频片段 |

**组织方式**：
- 默认按关联的主题分组展示
- 顶部筛选栏：按类型筛选（全部/作品/观察/反思/照片/视频）
- 每条记录显示：类型图标、标题、上传时间、上传者

**数据存储**：

IndexedDB 表 `implementation_records`：
```
{
  id, planId, theme, kinderId,
  type: 'artwork'|'observation'|'reflection'|'photo'|'video',
  title, description,
  fileData: '',           // Base64（图片/视频）
  fileType: '',           // MIME type
  createdBy, createdAt
}
```

### 6.5 方案下载

- **下载PDF**：调用 `window.print()` 触发浏览器打印 → 另存为PDF
- **下载Word**：生成纯文本版本，复制到剪贴板或导出为 .doc（HTML格式保存为.doc）

## 七、技术实现

### 7.1 文件结构

```
index.html              → 欢迎页 + 登录页
app.html                → 主应用（三版块）
main.js                 → 核心逻辑（保留现有 + 新增）
styles.css              → 样式（保留现有 + 新增）
db.js                   → IndexedDB 封装（新增）
auth.js                 → 登录/注册逻辑（新增）
```

### 7.2 技术栈

- HTML + CSS + JS（纯前端，无框架）
- IndexedDB（数据持久化）
- 无外部依赖

### 7.3 IndexedDB 数据库

数据库名：`yuanbenkecheng_db`

| 表名 | 用途 |
|------|------|
| `kinders` | 园所信息 |
| `accounts` | 账号信息 |
| `curriculum_plans` | 园本课程方案 |
| `course_contents` | 课程内容（教案/资源包） |
| `implementation_records` | 实施记录 |
