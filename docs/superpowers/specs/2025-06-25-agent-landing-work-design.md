# 9位AI专家落地工作体系设计

> 版本：v1.0 | 日期：2025-06-25 | 状态：已确认

---

## 一、设计目标

为园本课程生成平台的9位AI专家定义完整的落地工作体系，确保：
1. 每位专家输出细化到可操作级别，减少教师二次编辑
2. 每步产出需经园长/教师人工确认后方可进入下一环节（确认门控）
3. 当有异议时，AI专家能与真人协作探讨，给出专业可落地的修订方案

---

## 二、专家分类与协作模式

| 类型 | 专家 | 协作模式 | 确认重点 |
|------|------|----------|----------|
| **诊断型** | Agent1 园所诊断师、Agent2 政策研究员、Agent3 地域文化挖掘师 | 人提供素材 → AI深度加工 | 诊断结论是否准确 |
| **创意型** | Agent4 特色定位、Agent5 课程架构师、Agent6 课程地图设计师 | AI主导生成 → 人审核选择 | 方案是否符合园所实际 |
| **评审型** | Agent7 课程实施、Agent8 课程评价、Agent9 评审委员会 | AI生成方案 → 人机对话打磨 | 方案是否可落地执行 |

---

## 三、每位专家的6维定义

每位专家按以下6个维度定义落地工作：

| 维度 | 说明 |
|------|------|
| **输入** | 该专家需要什么数据/素材才能开始工作 |
| **方法论** | 用什么专业框架/流程来分析 |
| **工具模板** | 交付什么可操作的工具（量表、表格、模板） |
| **产出物** | 最终交付什么成果 |
| **确认门控** | 哪些产出需要园长/教师确认，确认什么 |
| **异议协作** | 当有异议时，AI如何与真人协作（选项式/对话式/批注式） |

## 四、Agent 1 · 园所诊断师

| 维度 | 内容 |
|------|------|
| **输入** | 园所基础信息（表单已收集）+ 教师课程能力自评问卷 + 现有课程资料上传（教案、周计划、特色活动方案） |
| **方法论** | SWOT分析框架 + 课程成熟度5级模型 + 教师课程能力4维评估 |
| **工具模板** | ①《教师课程能力自评表》（10题量表，诊断师自动计分）②《课程现状SWOT分析矩阵》③《课程成熟度雷达图》（5维可视化）④《诊断访谈提纲》（供园长与教师座谈用） |
| **产出物** | 《课程现状分析报告》（含SWOT矩阵、成熟度等级、改进优先级排序） |
| **确认门控** | 园长确认：①SWOT分析是否准确 ②成熟度等级是否认可 ③改进优先级是否合理 |
| **异议协作** | 园长可逐项调整SWOT条目（如"我们其实有XX优势没被识别"），诊断师基于新信息重新分析；成熟度等级可上调/下调并附理由 |

## 五、Agent 2 · 政策研究员

| 维度 | 内容 |
|------|------|
| **输入** | 园所所在省市 + 园所等级（省级示范/市级示范/一级园等）+ 近期迎评计划（如"明年参评省示范"） |
| **方法论** | 政策文本分析 + 关键词匹配 + 政策契合度评估矩阵 |
| **工具模板** | ①《政策关键词对照表》（园所课程 vs 政策要求逐条对照）②《政策风险排查清单》（是否踩红线，如小学化倾向）③《迎评准备清单》（针对示范园评审的专项准备） |
| **产出物** | 《政策解读与课程建设建议》（含政策契合度评分、风险提示、迎评建议） |
| **确认门控** | 园长确认：①省市政策是否覆盖完整 ②风险排查是否遗漏 ③迎评建议是否可行 |
| **异议协作** | 园长补充本地特殊政策要求，研究员纳入分析；对话式讨论政策落地的具体策略 |

## 六、Agent 3 · 地域文化挖掘师

| 维度 | 内容 |
|------|------|
| **输入** | 园所周边资源（表单已收集）+ 园所所在区县的文化志/地方志摘要 + 教师已知的社区资源清单 |
| **方法论** | 资源分类法（自然/文化/产业/社区/非遗5类）+ 教育价值评估矩阵 + 课程转化路径设计 |
| **工具模板** | ①《地域资源地图》（可视化标注周边300m/1km/3km资源）②《资源教育价值评估表》（安全性、适宜性、可操作性、持续性4维评分）③《资源→课程转化卡片》（每种资源对应3-5个活动idea） |
| **产出物** | 《地域资源挖掘与课程转化报告》（含资源地图、价值评估、转化建议） |
| **确认门控** | 园长+教研主任确认：①资源清单是否完整 ②价值评估是否准确 ③转化建议是否实用 |
| **异议协作** | 教师补充AI未识别的本地资源，挖掘师补充转化方案；选项式选择优先利用的资源 |

## 七、Agent 4 · 特色定位专家组（多专家竞争）

| 维度 | 内容 |
|------|------|
| **输入** | Agent1诊断结论 + Agent2政策方向 + Agent3资源清单 + 园所已有品牌资料（园徽、园歌、宣传册等） |
| **方法论** | 4专家并行竞争 → 裁判综合评审 → 园长最终决策 |
| **工具模板** | ①《课程定位方案对比表》（4专家方案横向对比：课程名称/口号/主张/特色/适用场景）②《课程命名评分卡》（辨识度、传播力、教育性、文化性4维评分）③《品牌定位画布》（目标受众、核心价值、差异化优势、传播策略） |
| **产出物** | 《课程定位方案》（含课程名称、课程主张、课程口号、品牌定位、招生亮点） |
| **确认门控** | 园长+核心团队确认：①课程名称是否喜欢 ②口号是否朗朗上口 ③定位是否准确。需逐项确认 |
| **异议协作** | 园长可要求"换一批方案"或指定方向（如"口号要更诗意一些"），4专家重新生成；选项式提供2-3个备选名称/口号供快速决策；对话式深入讨论定位理念 |

## 八、Agent 5 · 课程架构师（多专家协同）

| 维度 | 内容 |
|------|------|
| **输入** | Agent4确定的课程定位 + Agent1-3的全部输出 + 园所现有课程资源（周计划、主题方案、特色活动方案） |
| **方法论** | 3专家协同分工：教育学专家→理念层 → 课程专家→目标层 → 课程设计专家→结构层，层层递进 |
| **工具模板** | ①《课程理念金字塔》（核心理念→分支理念→实践原则）②《课程目标体系表》（总目标→领域目标→年龄段目标→主题目标，四级分解）③《课程框架图》（基础课程60%+特色课程30%+拓展课程10%）④《课程模块说明书》（每个模块的名称、目标、内容、实施建议） |
| **产出物** | 《课程架构方案》（含课程理念、课程目标体系、课程框架、课程模块） |
| **确认门控** | 园长+教研主任确认：①课程理念是否认同 ②目标体系是否合理 ③课程框架比例是否合适 ④模块划分是否清晰 |
| **异议协作** | 教研主任可调整模块比例（如"我们基础课程想占70%"），架构师重新配比；对话式讨论理念表述，AI提供3种理念表述风格（学术型/诗意型/务实型）供选择 |

## 九、Agent 6 · 课程地图设计师

| 维度 | 内容 |
|------|------|
| **输入** | Agent5课程架构方案 + 园所学年校历 + 各年龄段幼儿发展特点 |
| **方法论** | 螺旋进阶模型（感知体验→探索发现→表达创造）+ 主题网络设计法 + 学年时间轴规划 |
| **工具模板** | ①《三年进阶路径总表》（小班/中班/大班 × 各维度对比）②《学期主题网络图》（每个主题的中心概念+分支活动+领域覆盖）③《学年课程日历》（按月规划主题、节日活动、特色活动）④《主题资源包清单》（每个主题需要的绘本、材料、场地、社区资源） |
| **产出物** | 《课程进阶地图》（含三年进阶路径、学期主题规划、学年课程日历、主题资源包） |
| **确认门控** | 教研主任+年级组长确认：①进阶路径是否符合幼儿发展规律 ②主题选择是否合适 ③时间安排是否合理 ④资源是否可获取 |
| **异议协作** | 年级组长可替换/增减主题（如"秋天主题我们想换成丰收节"），地图师调整主题网络；选项式提供替代主题方案；对话式讨论主题深度和广度 |

## 十、Agent 7 · 课程实施专家组（4专家协同）

| 维度 | 内容 |
|------|------|
| **输入** | Agent5课程架构 + Agent6课程地图 + 园所现有场地平面图 + 现有作息时间表 + 现有区域材料清单 + 现有家园共育制度 |
| **方法论** | 4专家并行协同：环境创设→物理空间 + 一日生活→时间流程 + 区域活动→材料玩法 + 家园共育→协同机制 |
| **工具模板** | ①《户外/室内环境改造方案图》（功能分区标注 + 材料投放清单 + 改造优先级）②《优化后一日作息表》（含特色课程环节嵌入点）③《区域材料投放清单》（常规区+特色区，含材料名称、玩法建议、发展价值）④《家园共育年度计划表》（月度活动安排 + 家长参与方式 + 预期效果）⑤《课程实施教师培训计划》（培训主题、时间、形式、考核方式） |
| **产出物** | 《课程实施方案》（含环境方案、一日流程、区域方案、家园共育方案、教师培训方案） |
| **确认门控** | 园长+后勤园长+年级组长确认：①环境改造是否可行（预算、空间）②一日流程是否合理 ③材料投放是否可获取 ④家园活动频次是否合适 |
| **异议协作** | 后勤园长可反馈"户外场地不够大，无法同时设4个区"，实施专家调整方案（如轮换使用）；选项式提供不同预算等级的环境方案（基础版/标准版/升级版）；对话式讨论一日流程的弹性空间 |

## 十一、Agent 8 · 课程评价专家组

| 维度 | 内容 |
|------|------|
| **输入** | Agent5课程目标体系 + Agent6课程地图 + Agent7实施方案 + 园所现有评价工具（如有） |
| **方法论** | 四维评价体系（儿童发展+教师成长+课程质量+家园共育）+ 嵌入式评价设计 + 评价闭环（观察→记录→分析→反馈→改进） |
| **工具模板** | ①《幼儿学习故事记录模板》（含观察要点、分析框架、支持策略）②《幼儿发展检核表》（分年龄段、分领域，可勾选+备注）③《教师课程反思日志模板》（含反思要点、改进计划）④《主题课程审议表》（含审议维度、评分标准、改进建议）⑤《学期课程质量自评量表》（含指标、评分、证据）⑥《家长满意度问卷模板》 |
| **产出物** | 《课程评价体系》（含评价理念、四维评价方案、全套评价工具模板、评价结果运用指南） |
| **确认门控** | 园长+教研主任确认：①评价维度是否全面 ②评价工具是否实用（会不会增加教师负担）③评价周期是否合理 |
| **异议协作** | 教研主任反馈"每周2篇学习故事太多了，老师写不过来"，评价专家调整频率和方式（如改为照片+一句话记录）；选项式提供轻量版/标准版/深度版评价方案；对话式讨论评价如何真正促进改进而非流于形式 |

## 十二、Agent 9 · 课程评审委员会

| 维度 | 内容 |
|------|------|
| **输入** | Agent1-8全部输出 + 园所自评报告（如有） |
| **方法论** | 4角色模拟评审（教育局专家+比赛评委+示范园专家+集团园专家）+ 五维打分（先进性、逻辑性、特色性、落地性、政策契合度）+ 回流优化闭环 |
| **工具模板** | ①《五维评审打分表》（含评分标准、权重、专家评语）②《课程方案修改对照表》（原方案→评审意见→修改方案→修改后评审）③《课程建设成果清单》（可参赛、可申报、可展示的成果梳理）④《回流优化任务看板》（按优先级排列的待优化项，含负责人、截止时间、完成状态） |
| **产出物** | 《课程评审报告》（含五维打分、专家评语、修改建议、回流优化任务清单、成果申报建议） |
| **确认门控** | 园长+全体教师确认：①评审意见是否接受 ②修改优先级是否认可 ③回流优化任务分配是否合理 |
| **异议协作** | 园长可对评分提出异议（如"我们觉得特色性应该更高"），评审委员会重新评估并说明理由；对话式逐条讨论修改建议的可行性；选项式提供"立即优化/本学期优化/下学期优化"三级优先级方案 |

## 十四、异议协作的三种交互形式

| 形式 | 适用场景 | 交互方式 |
|------|----------|----------|
| **选项式** | 简单决策（选名称、选风格、选优先级） | AI提供2-3个替代方案，用户点击选择 |
| **对话式** | 深度探讨（理念讨论、方案调整） | 用户输入反馈，AI以对话形式回应，多轮交流后给出修订版 |
| **批注式** | 精准修改（具体段落/条目的调整） | 用户在具体内容上添加批注，AI逐条回应并修改 |

## 十五、待实现功能清单

基于以上设计，平台需要新增以下功能模块：

### 确认门控系统
- [ ] 每位Agent输出后显示"确认面板"
- [ ] 逐项确认（勾选 + 可选备注）

- [ ] 选项式：AI提供替代方案卡片，用户点击选择
- [ ] 对话式：聊天输入框，多轮对话

- [ ] 每位Agent附带可下载/可填写的工具模板
- [ ] 模板预览 + 下载按钮
- [ ] 在线填写（可选）

### 流程优化
- [ ] Agent1-3可并行执行（诊断型专家互不依赖）
- [ ] 每步确认后才能进入下一步
- [ ] 确认状态持久化
